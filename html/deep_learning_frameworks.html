深層学習フレームワーク

Define and Run と Define by Run

Define and Run

概要

フレームワークの考え方の一つで、計算グラフを構築してからデータを流す方式。

利点

最適化が行いやすい。

欠点

デバッグがしにくく、データ毎にネットワーク構造を変えにくい。

Define by Run

概要

フレームワークの考え方の一つで、計算をしながら計算グラフを構築する方式。

利点

デバッグがしやすく、データ毎にネットワーク構造を変更できる。

欠点

計算毎にネットワーク構築のコストがかかり、最適化を行いにくい。

具体例

Theano

pythonのDNNライブラリ。歴史的意義があるが、2017/11/15に1.0.0がでてから更新されていない。

Caffe

UCバークレーで開発された Deep Learning Framework。 Caffe Framework modelsが開発され、他のフレームワークでも使用できる Define and Run方式を採用。

Torch

Lua言語をベースとしたスクリプト言語の機械学習ライブラリ。2018年からは開発は行われていない

pyTorch

Pythonで使用できるTorch。2019年に開発された。 Define by Run方式を採用。

chainer

preferred network社が開発している和製フレームワーク。 Define by Run方式を採用。 2019/12/05 preferred network社はPyTorchに研究開発基板を移行を発表。 https://preferred.jp/ja/news/pr20191205/

TensorFlow

googleが開発、圧倒的に人気がある。Define and Run方式を採用。

keras

tensor flow, theanoのフロントエンド。人気がある

caffe2

caffeの改良版

TensorFlow.js

TensorFlowのjava script版
